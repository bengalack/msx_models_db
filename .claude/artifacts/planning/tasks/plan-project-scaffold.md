## Context
- Intended outcome: A working, runnable skeleton for both the web page (Vite + TypeScript) and the Python scraper, with correct directory structure, tooling, and command surface — so all subsequent feature work has a clean foundation to build on.

## Functional Snapshot
- Problem: The repository is empty. No tooling, no structure, no runnable commands. Nothing can be built or tested until the scaffold is in place.
- Target user: bengalack (developer/maintainer) setting up the project locally
- Success criteria (observable):
  - `npm run dev` starts the Vite dev server without errors
  - `npm run build` produces output in `docs/` (index.html + bundle.js)
  - `npm run typecheck` passes (zero TypeScript errors)
  - `npm run lint` passes (zero ESLint errors)
  - `npm test -- --run` exits 0 (no test failures; 0 tests is acceptable at this stage)
  - `python -m scraper` runs without error and prints a usage/placeholder message
  - Opening `docs/index.html` in a browser (file:// protocol) renders a placeholder page without console errors
- Primary user flow:
  1. Clone the repo
  2. Run `npm install` and `pip install -r requirements.txt`
  3. Run `npm run dev` and confirm dev server starts
  4. Run `npm run build` and confirm `docs/` is populated
  5. Open `docs/index.html` locally and confirm it loads
  6. Run `python -m scraper` and confirm it starts without error
- Must never happen:
  - Build step writes output outside `docs/`
  - Any hardcoded path that only works on one OS (Windows vs Unix)
  - `npm run build` or `python -m scraper` exits non-zero on a clean install
- Key edge cases:
  - Running `npm run build` twice → `docs/` is cleanly overwritten, no stale artefacts
  - `docs/index.html` opened via `file://` (no server) → renders without errors (no fetch calls in the stub)
- Business rules:
  - All colors in CSS must use `var(--...)` tokens (no hardcoded hex in component styles)
  - Output directory is always `docs/` — never `dist/`
  - `docs/data.js` is NOT generated by Vite; it is written by the scraper separately
- Integrations: none (scaffold only; no external calls)
- Non-functional requirements:
  - Privacy/Security: No secrets or credentials committed; `.env` in `.gitignore`
  - Performance: N/A for scaffold
  - Reliability: All commands reproducible on a fresh clone
- Minimal viable increment (MVI): All five `npm run *` commands and `python -m scraper` pass on a fresh install. `docs/index.html` opens locally.
- Deferred:
  - Data schema and seed data (next backlog item)
  - Theme CSS custom properties (next after data)
  - Any real application logic
  - GitHub Actions CI
  - GitHub Pages configuration (requires repo settings — documented but not automated here)
