# Decision Log

| Date | Decision | Why | Tradeoff |
| --- | --- | --- | --- |
| 2026-03-27 | Visual theme: CRT phosphor green (dark default) + warm cream (light) | MSX enthusiast audience; retro aesthetic fits the subject matter | More distinctive than generic enterprise; theme must remain swappable via CSS custom properties |
| 2026-03-27 | All colors via CSS custom properties on `[data-theme]`; zero hardcoded hex in components | Maintainer wants to be able to change the theme later without touching component code | Slightly more CSS boilerplate upfront |
| 2026-03-27 | Monospace font throughout (Share Tech Mono / Consolas fallback) | Reinforces terminal aesthetic; ensures column alignment in dense data grid | Less typographic flexibility than mixed font stack |
| 2026-03-27 | Compact row density (24px rows) | Grid has many columns and rows; desktop-first users expect dense data tools | Smaller touch targets — acceptable since mobile-first is out of scope |
| 2026-03-27 | Single-page layout; no modals or navigation | Grid is the only view; all controls are in-page toolbar or context menus | No room for detail panels or per-model views in this iteration |
| 2026-03-27 | Scraper language: Python 3.11+ (beautifulsoup4 + lxml) | Best-in-class HTML/XML scraping libraries; maintainer preference | Two runtimes in the project (Node.js for web, Python for scraper) |
| 2026-03-27 | Web page: Vanilla TypeScript + Vite (no framework) | Custom cell-selection model doesn't fit standard table library abstractions; Vite output is plain static files | More grid implementation work; Node.js required for development |
| 2026-03-27 | Build output committed to docs/ on main branch | GitHub Pages serves from docs/; no separate deploy pipeline needed | Git history includes build artifacts; diffs noisy after scraper runs |
| 2026-03-27 | URL state: versioned binary codec, base64 in hash fragment | Compact enough for large selections; hash has no hard length limit; no server involved | Not human-readable; requires careful version migration if format changes |
| 2026-03-27 | docs/data.js loaded via script tag (not fetch) | Works on file://, HTTP, and Blogger embed equally | Data is bundled as a JS assignment (window.MSX_DATA); slightly less clean than JSON |
| 2026-03-30 | Slot map LUT is a separate maintainer-edited JSON file (`data/slotmap-lut.json`) | New device strings will surface as more XML files are parsed; maintainer must extend vocabulary without touching scraper code | Extra file to validate on load; not self-documenting without comments |
| 2026-03-30 | Slot map classification uses XML element type + `id` attribute regex (not flat string matching) | openMSX XML is structured; element type alone disambiguates most devices (MemoryMapper vs PanasonicRAM vs ROM); `id` regex handles ROM subtypes | LUT rules must be tested in order; first-match-wins semantics must be documented clearly |
| 2026-03-30 | Mirror detection implemented in three methods in order of increasing complexity: `rom_visibility` first, SHA1 file size second, `<Mirror>` element (two-pass) third | Staged delivery reduces risk; `rom_visibility` is fully self-contained in XML; SHA1 lookup adds local file I/O; `<Mirror>` requires a two-pass walk | Delivering in stages means some mirrors are missed until all three methods are live |
| 2026-03-30 | `systemroms/` ROM files not committed; only `all_sha1s.txt` index is tracked | ROM files are copyrighted; SHA1 index is sufficient for file-path lookup when files are present locally | Mirror detection method 2 (ROM size) is unavailable in CI; silently degrades rather than fails |
| 2026-03-30 | Slot map LUT shipped to browser as flat `{ abbr: tooltip }` map embedded in data.js | Tooltip strings stored once per abbreviation, not repeated per cell; reduces data.js size for 64 columns × N models | Browser must do a LUT lookup on hover rather than reading a per-cell tooltip value directly |
| 2026-04-01 | Cell truncation applied as character-count clip in `buildDataRow`; full value stored in `data-full-value` DOM attribute | Clipboard copy and sort both read `model.values[]` directly, so no changes needed there; `data-full-value` is the single source for tooltip and any future consumers | Per-cell DOM attribute overhead is negligible for 2 columns; character-count truncation may differ from pixel overflow but is deterministic and consistent across fonts |
| 2026-04-01 | For link cells with truncation, combined tooltip is `"<full value> — <url>"` written to `a.title` at render time | Single write at render time; mouseenter handler already skips link cells; no second code path needed | Tooltip format is fixed; cannot be changed without re-rendering the affected cells |
| 2026-04-16 | Scraper exclude list uses post-parse model matching + pre-fetch filename matching; fails fast with `ValueError` on malformed input; absent file is a no-op | Fail-fast on bad config prevents silent data loss; absent-is-no-op keeps the feature optional with zero setup cost | Case-sensitive matching only; regex/glob patterns beyond `"*"` deferred |
| 2026-04-16 | RTC column populated from openMSX XML via `<RTC>` tag presence under `<devices>`; msx.org-only models receive `null` (not `"No"`) | `"No"` means confirmed absent; `null` means unknown — the distinction matters for data quality and avoids misleading collectors | Requires XML data to be meaningful; models without XML appear with an empty RTC cell |
| 2026-04-16 | Column cell shading uses a `shaded` boolean flag on `ColumnDef`; applied as a CSS class on `<td>` only; derived programmatically from column key pattern (not a hard-coded ID list) | Separates display from data; pattern-based derivation means new slot map columns get correct shading without manual config | Shading is binary — no gradient or intensity levels; sufficient for current use case |
