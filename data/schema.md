# MSX Models DB — Data Schema

## Overview

All model data is stored in `docs/data.js`, which sets `window.MSX_DATA` when loaded via a `<script>` tag. The TypeScript types are defined in `src/types.ts`.

The file is generated by the scraper (`python -m scraper`) and committed to the repository. A hand-curated seed version lives in `public/data.js` (Vite copies it to `docs/data.js` on each build; the scraper overwrites it after a scraper run).

---

## MSXData (root object)

| Field | Type | Description |
|---|---|---|
| `version` | `number` | Schema version integer. Currently `1`. Increment on breaking changes. |
| `generated` | `string` | ISO date (`YYYY-MM-DD`) of when the file was generated. |
| `columns` | `ColumnDef[]` | All column definitions, in display order. |
| `groups` | `GroupDef[]` | All group definitions. |
| `models` | `ModelRecord[]` | All model records. |
| `defaultView` | `DefaultViewConfig?` | Optional default view. Omit to fall back to show-all. |

---

## ColumnDef

| Field | Type | Description |
|---|---|---|
| `id` | `number` | **Stable integer ID.** Permanently assigned. Never reassigned or reused. |
| `key` | `string` | Machine-readable key (snake_case). Matches `IDRegistry.columns`. |
| `label` | `string` | Display label shown in the column header. |
| `groupId` | `number` | ID of the `GroupDef` this column belongs to. |
| `type` | `'string' \| 'number' \| 'boolean'` | Data type of values in this column. |

## ModelRecord

| Field | Type | Description |
|---|---|---|
| `id` | `number` | **Stable integer ID.** Permanently assigned. Never reassigned or reused. |
| `values` | `(string \| number \| boolean \| null)[]` | Field values, positionally aligned with `MSXData.columns[]`. |

### values[] alignment rule

`model.values[i]` corresponds to `MSXData.columns[i]`. The arrays **must** be the same length. If a value is unknown or not applicable, use `null` — never omit the entry or use `undefined`.

## IDRegistry (`data/id-registry.json`)

The ID registry is maintained by the scraper and committed to the repository. It is **not** shipped to the browser.

| Field | Type | Description |
|---|---|---|
| `version` | `number` | Registry format version. Currently `1`. |
| `models` | `Record<string, number>` | Natural key → stable model ID. Natural key: `"<manufacturer>\|<model>"` (lowercase, trimmed). |
| `retired_models` | `number[]` | Model IDs retired when a model is removed. Never reused. |
| `next_model_id` | `number` | Next ID to assign for a new model. Only increases. |
| `columns` | `Record<string, number>` | Column key → stable column ID. Pre-assigned once; never changes. |
| `next_column_id` | `number` | Next ID to assign for a new column. Only increases. |

### Stable ID contract

- IDs are short positive integers starting at 1.
- Once assigned, an ID is **permanent** — it identifies the same entity forever.
- Retired IDs are recorded in `retired_models` but **never reused**.
- `next_model_id` and `next_column_id` only ever increase.
- The scraper must attempt to **match before create**: match incoming data to an existing registry entry (by natural key) before assigning a new ID.

## Versioning

The `MSXData.version` field enables future schema evolution:

- **Additive changes** (new columns with new IDs): no version bump required.
- **Breaking changes** (column removed, type changed, values[] alignment changed): bump `version`.
- The URL codec has its own version byte (independent of `MSXData.version`). Both are checked at decode time and fall back to defaults on unknown versions.
