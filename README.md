# MSX Models DB

A single static web page presenting MSX2, MSX2+, and MSX turbo R computer model specifications in a sortable, filterable, spreadsheet-like grid. All state (sort, filters, hidden rows/columns, cell selection) is encoded in the URL hash so any view can be bookmarked or shared. Works on `file://`, GitHub Pages, and as a Blogger/Blogspot embed.

## Prerequisites

- [Node.js](https://nodejs.org/) 20 or later
- [Python](https://www.python.org/) 3.11 or later

## Setup

```sh
# Install JavaScript dependencies
npm install

# Install Python dependencies (use a virtual environment)
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Commands

| Command | Description |
|---|---|
| `npm run dev` | Start Vite dev server at `http://localhost:5173` |
| `npm run build` | Build production output to `docs/` |
| `npm run typecheck` | Run TypeScript type checks (no emit) |
| `npm run lint` | Run ESLint on `src/` |
| `npm test -- --run` | Run Vitest test suite once |
| `python -m scraper` | Run the MSX data scraper |

## GitHub Pages

The `docs/` directory is committed to the repository and served directly by GitHub Pages.

To enable GitHub Pages:

1. Go to **Settings → Pages** in the repository.
2. Set **Source** to `Deploy from a branch`.
3. Set **Branch** to `main` and **Folder** to `/docs`.
4. Save. The page will be available at `https://<username>.github.io/<repo>/`.

> The `docs/data.js` file (generated by the scraper) must be hosted on GitHub Pages for the Blogger/Blogspot embed to load data correctly. The CORS headers served by GitHub Pages allow cross-origin script loads.
